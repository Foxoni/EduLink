================================================================
  EduLink — Explication de l'application
================================================================

DESCRIPTION GENERALE
---------------------
EduLink est une application web de gestion scolaire développée
en Python avec le framework Flask. Elle permet à un administrateur
de gérer les classes, les utilisateurs et les emplois du temps
d'un établissement scolaire.

L'application tourne dans Docker et utilise une base de données
MySQL pour stocker toutes les données.


TECHNOLOGIES UTILISEES
-----------------------
- Backend  : Python 3.9 + Flask 3.0
- Base de données : MySQL 8.0 (via mysql-connector-python)
- Templates : Jinja2 (moteur de templates intégré à Flask)
- CSS : Design system maison (static/css/style.css)
- Icônes : Bootstrap Icons (CDN)
- Polices : Google Fonts (Poppins + Roboto)
- Conteneurisation : Docker + Docker Compose
- Sécurité : Flask-WTF (CSRF), bcrypt (hachage des mots de passe)


LANCER L'APPLICATION
---------------------
Avec Docker (recommandé) :

  docker-compose up -d
  docker exec gcs_flask_app python seed.py   <- injecte les données de test

  Accès :
    Application  -> http://localhost:5001
    PHPMyAdmin   -> http://localhost:8080

Comptes de test générés par seed.py :
  Admin      : admin        / Admin1234!
  Professeur : prof.martin  / Prof1234!
  Elève      : (voir credentials.txt après seed)


STRUCTURE DES FICHIERS
-----------------------
admin.py               -> Point d'entrée principal (lancé par Docker)
                          Contient toutes les routes de l'espace admin :
                          /classes, /utilisateurs, /emploi-du-temps, etc.

db.py                  -> Fonction de connexion à la base MySQL
decorators.py          -> Décorateurs @login_required et @role_required
                          pour protéger les routes selon le rôle
seed.py                -> Peuple la base avec des données de test
init.sql               -> Schéma SQL de la base de données

blueprints/            -> Architecture modulaire par rôle (app.py)
  auth/                -> Connexion / déconnexion
  admin/               -> Tableau de bord admin (blueprint)
  prof/                -> Espace professeur
  eleve/               -> Espace élève

templates/
  login.html           -> Page de connexion
  admin/
    layout.html        -> Squelette HTML partagé par les 3 pages admin
                          (sidebar, head, alertes flash)
    classe.html        -> Gestion des classes (création, liste)
    utilisateur.html   -> Gestion des utilisateurs (ajout, liste, suppression)
    emploi.html        -> Emploi du temps (ajout de cours, planning)
  prof/
    dashboard.html     -> Tableau de bord professeur
  eleve/
    dashboard.html     -> Tableau de bord élève

static/css/style.css   -> Feuille de style unique pour toute l'application


BASE DE DONNEES — TABLES PRINCIPALES
--------------------------------------
Roles            -> Les 3 rôles : Admin (1), Professeur (2), Elève (3)
Classes          -> Les classes de l'établissement (ex: 2nde A, Tle C)
Utilisateurs     -> Tous les utilisateurs dans une seule table.
                    Champ id_role pour distinguer le rôle.
                    Champ matiere uniquement pour les profs.
                    Champ id_classe uniquement pour les élèves.
Prof_Classe      -> Table de liaison Professeur <-> Classe (plusieurs à plusieurs)
Emploi_du_temps  -> Planning des cours (classe, prof, date, heures, salle)
Evaluation       -> Devoirs/contrôles créés par les profs
Notes            -> Notes des élèves liées aux évaluations


ESPACE ADMIN — FONCTIONNALITES
--------------------------------
Page Formations (/classes) :
  - Créer une nouvelle classe
  - Voir toutes les classes existantes sous forme de cartes
  - Cliquer sur une classe pour accéder à son emploi du temps

Page Utilisateurs (/utilisateurs) :
  - Ajouter un utilisateur (Admin, Professeur ou Élève)
  - Si Professeur : saisir la matière enseignée
  - Si Élève : assigner une classe
  - Modifier la classe d'un élève directement dans le tableau
  - Supprimer un utilisateur

Page Planning (/emploi-du-temps) :
  - Sélectionner une classe pour voir son emploi du temps
  - Ajouter un cours (professeur, date, heures, salle)
  - Détection automatique des conflits d'horaires


SECURITE
---------
- Les mots de passe sont hachés avec bcrypt avant stockage
- Toutes les requêtes SQL utilisent des paramètres (%s)
  pour éviter les injections SQL
- Protection CSRF via Flask-WTF sur tous les formulaires POST
- Sessions avec durée de vie limitée (1 heure)
- En-têtes de sécurité HTTP ajoutés automatiquement :
    X-Frame-Options: DENY
    X-Content-Type-Options: nosniff
    Content-Security-Policy
    Referrer-Policy


REMARQUE — DEUX POINTS D'ENTREE
---------------------------------
Le projet contient deux fichiers Flask distincts :

  admin.py  -> Version autonome utilisée par Docker.
               Gère les routes admin sans authentification.
               C'est ce fichier qui est lancé en production Docker.

  app.py    -> Version complète avec blueprints et authentification.
               Contient la gestion des sessions, login/logout,
               et les espaces professeur et élève.

Pour un déploiement complet (avec login), il faudra basculer
le Dockerfile pour lancer app.py au lieu de admin.py.
